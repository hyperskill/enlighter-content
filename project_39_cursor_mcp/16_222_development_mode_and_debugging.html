Now that we've built and tested our Echo MCP server, let's explore the development and debugging features provided by the MCP Python SDK. These tools will help us troubleshoot issues and develop more complex MCP servers with ease.

Understanding Development Mode

The MCP Python SDK includes a powerful development mode that provides a web interface for testing and debugging your MCP servers. This interface allows you to:

- Inspect server capabilities and configurations
- Test resources, tools, and prompts interactively
- View request and response details
- Debug communication between clients and your server

Development mode is especially useful when building complex MCP servers, as it provides immediate feedback without requiring a full integration with client.

Running the Server in Development Mode

Let's run our Echo MCP server in development mode:

Run 'mcp dev echo_server.py' in terminal to start the Echo MCP server in development mode. Don't forget to use our virtual environment.

This will start the server and open a web interface for debugging.

Now you can see the debug interface in your browser by opening the link shown in the terminal output (typically http://localhost:5173).

Connect to your MCP server by clicking the "Connect" button in the interface.

Make sure your MCP server is running and you're connected to it in the Inspector interface. Inspecting Resources

One of the most powerful features of the MCP Inspector is the ability to explore and test the resources exposed by your server. Let's see how to inspect resources:

In the MCP Inspector interface, click the "Resources" tab and then click the "List Resources" or "List Templates" button.
This will send a request to your server to retrieve all available resource patterns.
You should see echo_resource in the results.

Inspecting Tools

The MCP Inspector also makes it easy to explore and test the tools exposed by your server. Here's how to inspect tools:

Click the "Tools" tab and then click the "List Tools" button in the interface.
This sends a request to retrieve all available tools.
You should see "echo_tool" in the results.

When a tool is listed, you can click on tool name and can see messages texbox. It is a tool input field. AI uses this field when calls the tool.

To test our echo tool:
- Enter a test message: Testing echo tool from Inspector
