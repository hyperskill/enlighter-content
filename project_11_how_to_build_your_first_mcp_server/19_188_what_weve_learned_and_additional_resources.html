Bonus: Browser-Use MCP

As a bonus, let's create one more MCP server that allows the AI to use a web browser directly. This will enable the AI to search the web, read content, and interact with websites on your behalf.

Click to expand and create the Browser-Use MCP Setting Up the Browser-Use MCP

First, let's create a new Python file for our browser-use MCP:

Create a file named browser_use_mcp.py with the following content: from mcp.server.fastmcp import FastMCP import subprocess import sys import json import os mcp = FastMCP("Browser Use") @mcp.tool() def use_browser(url: str = None, query: str = None) -> str: """Use a web browser to find information or interact with websites. Args: url: Optional URL to navigate to directly query: Optional search query to look up Returns: Information found or confirmation of action taken """ if not url and not query: return "Please provide either a URL to visit or a search query." # If only a query is provided, convert it to a search URL if query and not url: # Use DuckDuckGo for privacy url = f"https://duckduckgo.com/?q={query.replace(' ', '+')}" # Open the URL in the default browser if sys.platform == 'darwin': # macOS subprocess.run(['open', url]) elif sys.platform == 'win32': # Windows os.startfile(url) else: # Linux subprocess.run(['xdg-open', url]) return f"Opened browser with {'search query: ' + query if query else 'URL: ' + url}" if __name__ == "__main__": # Print to stderr so it doesn't interfere with MCP communication print("Browser Use MCP server starting...", file=sys.stderr) mcp.run(transport='stdio') Make sure the file is created with the correct content. Adding to MCP Configuration

Now, let's add our browser-use MCP to the Cursor configuration:

Update the .cursor/mcp.json file to include the browser-use MCP server (browser_use_mcp.py). Use the same format as for echo_server.py. The configuration must have 'Browser use' as the name of the MCP server. Make sure the configuration file is updated with the browser-use MCP server. Enabling in Cursor Settings
Make sure the browser-use MCP is enabled in Cursor settings. Sometimes you may note errors in MCP status in the settings. This may happen because browser-use uses standard output for logging. As we already know, MCPs use standard input/output for communication. You can experiment with config to disable logging. We leave it as an exercise for you. Testing our browser-use tool
