What is Memory Bank?

Memory Bank is a structured documentation system that helps GitHub Copilot maintain context across sessions. It transforms Copilot from a stateless assistant into a persistent development partner that can effectively "remember" your project details over time.

When working with AI coding, you've likely encountered this common limitation: the AI has no memory of your previous conversations once you start a new session. This means:

  • You need to re-explain your project repeatedly
  • The AI can't recall decisions you made together
  • You lose continuity in complex projects
  • Each session starts from scratch
How Memory Bank Solves This Problem

Memory Bank addresses this limitation through a systematic approach to documentation:

  • Structured Documentation: A hierarchy of markdown files that build a complete picture of your project.
  • Context Preservation: Key information is stored in a format that's easy for both you and Copilot to reference.
  • Continuous Updates: Documentation evolves as your project progresses.
Memory Bank is Not Magic

It's important to understand that Memory Bank isn't a magical feature that gives Copilot perfect memory. It's a methodical approach to documentation that you maintain that enhances Copilot's context management capabilities.

Setting Up Memory Bank

To set up Memory Bank we will use Memory Bank MCP Server. It's a simple way to create a Memory Bank for your projects automatically based on provided context.

First of all, we need to install uv and uvx tools to run Memory Bank MCP Server.

Install uv and uvx tool if it's not installed yet

Then, we need to configure Memory Bank MCP Server.

Enable the Memory Bank MCP Server

To configure an MCP server for a specific workspace, we should create a .vscode/mcp.json file in your workspace folder.

To save some time we can ask Copilot to create a mcp.json file with the right configuration for the MCP server we want to install. We will use it as the main way to configure MCP servers in our course.

Create or edit .vscode/mcp.json and add: { "servers": { "mcp-memory-bank": { "command": "uvx", "args": [ "--from", "git+https://github.com/ipospelov/mcp-memory-bank", "mcp_memory_bank" ] } } } If mcp.json file is already created, just add "mcp-memory-bank" object to the "servers" array and don't remove existing servers.

If everything went smoothly, you should see the following file created in your workspace folder by Copilot:

If you look closely, there is the menu above "mcp-memory-bank": line. Press Start button to enable the MCP server and it should say Running now.

Well done! You have installed Memory Bank MCP Server manually.

Limitations
MCP will work only in Agent mode. We recommend to use it throughout this lesson. Having too many MCP servers and tools added can confuse GitHub Copilot and make it less reliable in choosing correct tools. Make sure to only enable the MCP servers and tools that you really need for your current project.
