The Memory Bank MCP provides a structured way to maintain project context for AI assistants. By organizing project information into categorized markdown files, Memory Bank helps GitHub Copilot better understand your project's goals, architecture, and current focus. This stage will guide you through setting up and effectively utilizing the Memory Bank MCP in your GitHub Copilot workflow.

Features:
  • Organize project context in hierarchical markdown files
  • Maintain persistent knowledge between GitHub Copilot sessions
  • Retrieve specific context information on demand
  • Generate template files based on project needs
  • Analyze project summaries for better context organization

Key Use Cases
  • Long-term project development with consistent AI assistance
  • Complex projects requiring detailed context understanding
  • Team collaboration where contextual knowledge needs sharing
  • Projects with evolving requirements and architecture

Memory Bank organizes project context into six core markdown files:

  1. projectbrief.md
    • Foundation document with core requirements
    • Defines project scope, goals, and success criteria
    • Created first and rarely changes

  2. productContext.md
    • The "why" of your project
    • Explains user needs and problems being solved
    • Outlines user experience goals

  3. systemPatterns.md
    • The "how" of your project
    • Documents system architecture and design patterns
    • Records key technical decisions

  4. techContext.md
    • Tools and technologies
    • Lists frameworks, libraries, and development tools
    • Documents technical constraints

  5. activeContext.md
    • Current project state
    • Describes what you're working on now
    • Records recent changes and next steps

  6. progress.md
    • Project status tracking
    • Lists completed features and known issues
    • Records the evolution of project decisions

This MCP helps to automatically create memory bank files in your project.

Setup Instructions

To set up Memory Bank we will use Memory Bank MCP Server. It's a simple way to create a Memory Bank for your projects automatically based on provided context.

First of all, we need to install uv and uvx tools to run Memory Bank MCP Server.

Install uv and uvx tool if it's not installed yet

Then, we need to configure Memory Bank MCP Server.

Create or edit a file .vscode/mcp.json to include the following MCP setup for Memory Bank: { "servers": { "mcp-memory-bank": { "command": "uvx", "args": [ "--from", "git+https://github.com/ipospelov/mcp-memory-bank", "mcp_memory_bank" ] } } } If mcp.json file is already created, just add "mcp-memory-bank" to the "mcpServers" array.

Skip this section if you already have Memory Bank MCP Server configured with uvx.

Setup with Docker Create or edit a file .vscode/mcp.json to include the following MCP setup for Memory Bank: { "servers": { "memory-bank": { "command": "docker", "args": [ "run", "-i", "--rm", "19283744/mcp-memory-bank:latest" ] } } } If mcp.json file is already created, just add "mcp-memory-bank" to the "servers" array.
