What are MCP Servers?

MCP (Model Context Protocol) servers are modular tools that connect to your AI Agent and provide it with additional context and tools to achieve better results in writing code, debugging, and more.

What You'll Learn
  • How to install MCP servers in GitHub Copilot
  • Various types of MCP installations
  • Top MCP servers to boost your productivity and results in GitHub Copilot
  • How to find new MCP servers and tools for your needs
Prerequisites

Before setting up any MCP servers, you'll need to ensure you have the necessary tools installed on your system:

NPX Installation

Throughout this project, we'll primarily use NPX to install and run MCP servers. NPX is included with Node.js and npm, which you'll need to install if you don't have them already:

Install npx, node, and npm. Verify your installation by running: ``` node --version npm --version npx --version ``` All three commands should display version numbers if the installation was successful.
UV and UVX Installation

Some of MCP servers require uv and uvx tools to run. You can install them by running the following command:

Install uv and uvx tool if it's not installed yet
Docker (Optional)

Some MCP servers provide Docker installation options, which can be useful for containerized environments or if you prefer isolation. Throughout this project, we'll show Docker configuration examples alongside NPX options.

If you choose to use Docker, you'll need to install Docker Desktop or Docker Engine on your system. Installation instructions vary by operating system and are beyond the scope of this project. Please refer to the official Docker documentation for installation guidance.

Manual MCP Server Installation

Let's see how to install any MCP Server manually. We will use Memory Bank MCP Server as an example.

To configure an MCP server for a specific workspace, we should create a .vscode/mcp.json file in your workspace folder.

To save some time we can ask Copilot to create a mcp.json file with the right configuration for the MCP server we want to install. We will use it as the main way to configure MCP servers in our course.

Create or edit a file .vscode/mcp.json to include the following MCP setup for Memory Bank: { "servers": { "mcp-memory-bank": { "command": "uvx", "args": [ "--from", "git+https://github.com/ipospelov/mcp-memory-bank", "mcp_memory_bank" ] } } } If mcp.json file is already created, just add "mcp-memory-bank" to the "mcpServers" array.

If everything went smoothly, you should see the following file created in your workspace folder by Copilot:

If you look closely, there is the menu above "mcp-memory-bank": line. Press Start button to enable the MCP server and it should say Running now.

Well done! You have installed Memory Bank MCP Server manually.

To configure an MCP server for all your workspaces, you can add the server configuration to your user settings. This allows you to reuse the same server configuration across multiple projects.

Specify the server in the mcp VS Code user settings to enable the MCP server across all workspaces.

Limitations
MCP will work only in Agent mode. We recommend to use it throughout this lesson. Having too many MCP servers and tools added can confuse GitHub Copilot and make it less reliable in choosing correct tools. Make sure to only enable the MCP servers and tools that you really need for your current project.
