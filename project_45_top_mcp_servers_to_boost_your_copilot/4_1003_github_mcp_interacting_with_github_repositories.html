The GitHub MCP enables GitHub Copilot to interact directly with GitHub repositories, allowing you to manage your code, issues, pull requests, and more without leaving your GitHub Copilot environment. This powerful integration provides comprehensive features including file operations, repository management, advanced search functionality, and pull request workflows. This stage will guide you through setting up and utilizing this powerful GitHub integration tool.

Features:
  • Create or update files in repositories with automatic branch creation
  • Push multiple files in a single commit with batch operations support
  • Search repositories, code, issues, and users with advanced search capabilities
  • Manage repository issues and pull requests with comprehensive error handling
  • Review and merge pull requests while preserving Git history
  • View file and directory contents
  • Perform both single-file and multi-file operations efficiently

Key Use Cases
  • Creating and managing GitHub repositories
  • Accessing and modifying repository content
  • Working with issues and pull requests
  • Performing advanced GitHub searches
  • Automating repository operations

Setup Instructions

You can set up the GitHub MCP using either NPX or Docker. Both methods require a GitHub Personal Access Token.

To create your GitHub Personal Access Token, go to GitHub Settings → Developer Settings → Personal Access Tokens. For full repository access, select the `repo` scope; for public repositories only, select the `public repositories` scope.

NPX Setup
Create or edit a file .vscode/mcp.json to include the following MCP setup: { "servers": { "github": { "command": "npx", "args": [ "-y", "@modelcontextprotocol/server-github" ], "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "your_github_personal_access_token" } } } } If mcp.json file is already created, just add "github" object to the "servers" array and don't remove existing servers.
Docker Setup
Create or edit a file .vscode/mcp.json to include the following MCP setup: { "servers": { "github": { "command": "docker", "args": [ "run", "-i", "--rm", "-e", "GITHUB_PERSONAL_ACCESS_TOKEN" ], "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "your_github_personal_access_token" } } } } If mcp.json file is already created, just add "github" object to the "servers" array and don't remove existing servers.
