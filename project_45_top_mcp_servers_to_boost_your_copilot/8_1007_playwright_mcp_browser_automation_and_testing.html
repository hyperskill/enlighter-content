The Playwright MCP is a powerful server that brings Microsoft's Playwright browser automation capabilities to GitHub Copilot. This extensive toolkit enables precise control over web browsers, allowing for sophisticated web interactions, testing, and automation.

Features:
  • Comprehensive browser control with multiple interaction modes
  • Keyboard and mouse control for complex interactions
  • Screenshot capabilities with customization options
  • Tab and window management for multi-page workflows

Key Use Cases
  • Automated web testing and verification
  • Complex form filling and submission
  • Web scraping with precise element targeting
  • Multi-step workflow automation
  • Cross-browser compatibility testing

GitHub Copilot can handle a limited amount of tool slots. If you encounter a problem that new MCPs do not connect due to restrictions, turn off unnecessary ones to free up tool slots.
Setup Instructions

You can set up the Playwright MCP using NPX, which is the recommended method:

NPX Setup
Create or edit a file .vscode/mcp.json to include the following MCP setup: { "servers": { "playwright": { "command": "npx", "args": [ "@playwright/mcp@latest" ] } } } If mcp.json file is already created, just add "playwright" object to the "servers" array and don't remove existing servers.

For headless browser operation (no visible browser window), use:

Create or edit a file .vscode/mcp.json to include the following MCP setup: { "servers": { "playwright": { "command": "npx", "args": [ "@playwright/mcp@latest", "--headless" ] } } } If mcp.json file is already created, just add "playwright" object to the "servers" array and don't remove existing servers.
