Automating Browser Interactions with MCP and Playwright

A powerful application of MCP is browser automation for tasks like end-to-end testing.

By connecting Junie to the Playwright MCP server, you can delegate complex frontend testing scenarios to the agent, which it can execute and report on.

This approach is ideal for experienced developers who want to automate QA processes directly from their IDE. Instead of writing brittle test scripts, you can describe a user flow in natural language.

Let's see how this works.

Configuring the Playwright MCP Server

First, we need to add the Playwright MCP server to our project's configuration. This server is conveniently available as an NPX package, which simplifies the setup.

Ask Junie to create and populate the MCP configuration file for you.

Create a project-level MCP configuration file at `.junie/mcp/mcp.json` and configure the Playwright server in it: { "mcpServers": { "playwright": { "command": "npx", "args": [ "@playwright/mcp@latest" ] } } } This command uses `npx` to download and run the Playwright MCP server on the fly. Ensure you have Node.js and npm installed. If you are on Windows, Junie may need to configure the MCP server differently, via using cmd: { "mcpServers": { "playwright": { "command": "cmd", "args": ["/c", "npx", "-y", "@playwright/mcp@latest"] } } }

Once Junie creates the file, go to the settings and enable the MCP server. You can verify this in Settings → Tools → Junie → MCP Settings, where you should see the "playwright" server with a list of its available browser automation tools.

Use the prompt above and then verify in the Junie settings that the server is active.
Testing a User Flow with Junie and Playwright

Now, let's use this new capability to test a real-world user flow. Imagine we have a web application with a login form. We can ask Junie to perform the test for us.

For this example, let's use a public demo site. Let's ask Junie to test the login functionality on saucedemo.com.

Using the Playwright tools, go to `https://www.saucedemo.com`. Enter the username `standard_user` and the password `secret_sauce`. Click the login button. After logging in, verify that the page contains the text "Products". Report back on whether the test was successful.

Junie will now use the Playwright MCP tools to:

  1. Navigate to the URL.
  2. Locate the username and password fields and fill them in.
  3. Click the login button.
  4. Scan the resulting page for the verification text.
  5. Summarize the outcome in the chat.

This demonstrates how you can offload repetitive but critical frontend testing to the agent, freeing you up to focus on development.
