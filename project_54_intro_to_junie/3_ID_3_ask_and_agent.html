From Inquiry to Action

The core of working with Junie revolves around a powerful loop: using Ask mode to inquire and learn, and using Agent mode to plan and execute. Let's walk through a complete workflow where we first learn about a feature and then have the agent implement it for us.

Step 1: Inquire with 'Ask' Mode

Imagine you know that Junie can be customized, but you're not sure how. This is a perfect use case for 'Ask' mode. You can ask Junie about its own functionality. Let's find out about its project rules system.

What are Junie project rules? Explain their purpose, where the files should be stored, and what format they should use.

Junie will provide a detailed explanation. It will tell you that rules are stored in `.mdc` files inside a `.cursor/rules` directory, and that they are used to guide the agent's behavior to maintain project-specific standards. Now you have the knowledge.

Think of 'Ask' mode as your built-in documentation and pair-programming partner, always ready to clarify concepts or details about your code or the tool itself.
  • Use the prompt above to learn about the rules system directly from the AI.
Step 2: Plan and Execute with 'Agent' Mode

Now that we know how to create a rule, let's delegate the task of creating one to the Agent mode. We'll create a rule that enforces a header comment on all new Python files. Based on the information from 'Ask' mode, we can formulate a precise request.

Switch to Agent mode. Create a project rule for Python files. The rule should state that all new Python files must start with a header comment: `# Copyright (c) 2024 MyAwesomeProject`. Create the necessary directory and file (`.cursor/rules/python_headers.mdc`).

Junie will now create a plan:

  1. Check if the `.cursor/rules` directory exists.
  2. If not, create the directory.
  3. Create the file `python_headers.mdc` in that directory.
  4. Add the specified rule content to the file.

After you approve the plan, Junie will execute it. In moments, your project has a new rule, created correctly according to the feature you just learned about. You can then test this rule by asking the agent to create a new Python file and observing that the header is automatically included.

  • Use the prompt to have Junie create the rule file and directory.
  • Approve the plan and verify that the file and its content are created as expected.

This "Ask -> Act" cycle is fundamental to working effectively with Junie. You use conversation to resolve ambiguity and gain knowledge, then use the agent to perform the work with precision and speed.
