One of the most powerful aspects of Cursor Rules is that you can create them automatically using Cursor's AI capabilities. Let's explore this feature and learn about some advanced rule settings.

Improving Automatic Rule Creation

We've already seen how to create rules with Cursor's help. Now, let's take it a step further by creating a meta-rule - a rule about rules themselves!

Rules for Rules: Location Standards

We'll create a rule that ensures all rules are properly located in the .cursor/rules directory:

Create a new rule file called rules-location.mdc in .cursor/rules folder, add the following content: # Cursor Rules Location Locate all rules in .cursor/rules Apply this rule for every edit or create rule

Here we want to set the Rule Type for the file to “Always” as we want to apply this rule to every request. We recommend to set it manually in Cursor as the most reliable way to set the rule type.

When to Use "Always" (alwaysApply: true)

The alwaysApply setting is most useful for:

  • Critical project-wide standards
  • Fundamental architectural principles
  • Meta-rules (like our rules-location rule)

Use alwaysApply sparingly! Too many always-applied rules can slow down Cursor and create conflicting instructions.
Completion Checklist
  • Verify rules-location.mdc exists in .cursor/rules
  • Verify that rule type is set to "Always"
What We've Learned

In this stage, we've discovered some powerful capabilities of Cursor Rules:

  • We can create fine-grained rules for different purposes
  • We can create meta-rules that apply to other rules
  • We can use Rule Type "Always" for critical rules
  • We can automate rule creation using Cursor's AI

This demonstrates the flexibility and power of the rules system. You can create rules for specific file types, specific coding patterns, or even rules about how to manage other rules!
