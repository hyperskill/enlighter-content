The rules that apply within your project are described in files with the extension .mdc inside the .cursor/rules folder. You have already generated one such file in the previous steps. The applicability of these files within your project can be determined by defining Rule Type.

  • Always - rules are applied always and to any files in the project
  • Auto Attached - rules are applied only to files that fit the pattern specified in the settings
  • Agent Requested - whether to read or not to read the rule, whether to apply it within a particular request or not will be decided by the agent based on the description you specify in the rule. For example, if the description contains standards for the Python language, the agent will pay attention to these rules when creating files in this language
  • Manual - the agent will apply the rule only if you mention the rule in the request using the @ symbols

The Cursor interface hides the textual definition of these rules, but if you open an .mdc file in another editor, you will see options at the beginning of the file that specify these settings:

If the “alwaysApply” option is set to “true”, the Rule Type will be defined as “Always”. If you set “alwaysApply” to “false” and describe only the “description” field, the Rule Type will become “Agent Requested”. If you set “alwaysApply” to “false” and describe only the “globs” field, the Rule Type will become “Auto Attached”. If you set “alwaysApply” to “false” and describe all or none of the other options, the Rule Type will become “Manual”.

To set the Rule Type with a simple query to the agent, describe this mapping as a global rule for the cursor. In the User Rules input field inside Cursor Settings and inside the .cursorrules file, add this line:

When user tries to set cursor rule type in .cursor/rules/*.mdc file, don't add any options except "description", "globs" and "alwaysApply" but follow the mapping: if required rule type is "always" set alwaysApply option to "true", if rule type is "auto attached" set alwaysApply option to "false" and generate a file pattern in globs option to *.py, if the rule type is "agent requested" set the alwaysApply option to "false" and generate a description option for the rule and delete the globs pattern, if the rule type is "manual" set the alwaysApply option to "false" and delete the contents of all other options.

You can now manage Rule Types through the agent. For example, this prompt:

Set the Rule Type for the @coding-standards.mdc file to always

Or more appropriate for python language standards:

Set the Rule Type for the @coding-standards.mdc file to Auto Attached with pattern **/*.py

Using the setting from the second prompt allows the rule to be applied only to Python files.

If you need other files, you can list them too. Here are examples: - *.py - matches all Python files
- * - matches all files
- src/*.py - matches Python files in the src folder
- **/*.py - matches Python files in any folder
- test_*.py - matches Python files that start with "test_"
