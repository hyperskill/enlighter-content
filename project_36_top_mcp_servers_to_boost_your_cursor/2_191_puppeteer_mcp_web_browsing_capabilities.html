The Puppeteer MCP is a tool that provides browser automation capabilities using Puppeteer. This powerful server enables LLMs to interact with web pages, take screenshots, and execute JavaScript in a real browser environment. This stage will guide you through setting up and effectively utilizing this MCP in your Cursor workflow.

Features:
  • Navigate to any URL in the browser
  • Capture screenshots of the entire page or specific elements
  • Click elements and fill out forms on web pages
  • Hover over elements to trigger interactions
  • Execute JavaScript code in the browser console
  • Monitor browser console logs

Key Use Cases
  • Automate web testing and interaction
  • Extract data from websites
  • Fill out web forms programmatically
  • Capture visual evidence from websites
  • Debug web applications with console access

Setup Instructions

You can set up the Puppeteer MCP using either NPX or Docker. Choose the method that best suits your environment:

NPX Setup
Create or edit a file .cursor/mcp.json to include the following MCP setup: { "mcpServers": { "puppeteer": { "command": "npx", "args": [ "-y", "@modelcontextprotocol/server-puppeteer" ] } } }
Docker Setup
Create or edit a file .cursor/mcp.json to include the following MCP setup: { "mcpServers": { "puppeteer": { "command": "docker", "args": [ "run", "-i", "--rm", "--init", "-e", "DOCKER_CONTAINER=true", "mcp/puppeteer" ] } } }
