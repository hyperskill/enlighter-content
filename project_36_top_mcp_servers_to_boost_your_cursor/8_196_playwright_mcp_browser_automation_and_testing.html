We have already reviewed Puppeteer web automation, and Playwright is a more modern and feature-rich alternative.

The Playwright MCP is a powerful server that brings Microsoft's Playwright browser automation capabilities to Cursor. This extensive toolkit enables precise control over web browsers, allowing for sophisticated web interactions, testing, and automation.

Features:
- Comprehensive browser control with multiple interaction modes
- Keyboard and mouse control for complex interactions
- Screenshot capabilities with customization options
- Tab and window management for multi-page workflows

Key Use Cases
- Automated web testing and verification
- Complex form filling and submission
- Web scraping with precise element targeting
- Multi-step workflow automation
- Cross-browser compatibility testing

Cursor can handle limited amount of tools slots. If you encounter a problem that new MCPs do not connect due to restrictions, turn off unnecessary ones to free up tools slots. Setup Instructions

You can set up the Playwright MCP using NPX, which is the recommended method:

NPX Setup
Create or edit a file .cursor/mcp.json to include the following MCP setup: { "mcpServers": { "playwright": { "command": "npx", "args": [ "@playwright/mcp@latest" ] } } }

For headless browser operation (no visible browser window), use:

Create or edit a file .cursor/mcp.json to include the following MCP setup: { "mcpServers": { "playwright": { "command": "npx", "args": [ "@playwright/mcp@latest", "--headless" ] } } } Interaction Modes

Playwright MCP offers two primary interaction modes:

1. Snapshot Mode (Default)
- Uses accessibility snapshots for better performance
- More reliable for structured content
- Preferred for most automation tasks
2. Vision Mode
- Uses screenshots for visual-based interactions
- Better for visually complex interfaces
- Enable with the --vision flag:
Create or edit a file .cursor/mcp.json to include the following MCP setup: { "mcpServers": { "playwright": { "command": "npx", "args": [ "@playwright/mcp@latest", "--vision" ] } } } Practical Example: Web Automation
