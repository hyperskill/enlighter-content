Introduction

Now that you understand your role as a manager, it's time to learn your most important duty: creating a plan. Before you ask your AI to write a single line of code, you must first collaborate with it to build a project plan.

This single technique is arguably the most effective way to get better, more consistent results from your AI. A good plan acts as the project's long-term memory and guiding star, keeping both you and your AI intern focused and aligned.

The Challenge: The Intern with No Long-Term Plan

Imagine giving an intern a pile of bricks and saying, "build a house." Without a plan, they would start laying bricks randomly. The same is true for your AI. When you give it tasks without a master plan, you run headfirst into its core psychological weaknesses:

  • Amnesia Sets In: As we learned, your intern has a poor short-term memory (a limited context window). Without a written plan to refer back to, it will forget the project's goals and the features you've already built.
  • Bad Decisions Are Made: Lacking the "bigger picture," the AI will make poor architectural choices. It might build a feature in a way that conflicts with a future requirement that you haven't stated yet.
  • Effort is Wasted: Without clear milestones, you can't tell if you're making meaningful progress. You might spend hours on a feature only to realize it doesn't fit the overall goal, forcing you to scrap the work and start over.

Giving tasks without a plan isn't just inefficient; it's poor management. You are setting your talented intern up to fail.

The Solution: Creating Your Plan

The solution is to work with your AI to create a project plan before the coding begins. This process involves having the AI generate a formal project plan, which you then review and approve. This plan will can live in a `plan.md` file at the root of your project or in a special file depending on the IDE or AI agent you are using.

A good plan contains three critical sections:

  • The Goal: A clear description of what you're building. Project Requirements Document is a good example of this.
  • Context: Provide all important details about your product, such as the target audience, technical requirements, and limitations.
  • The Milestones: The project broken down into small, concrete, and testable steps. Each milestone is a mini-victory that moves you forward.
Pro Tip: Use the Right Model for the Planning

For high-level strategic thinking, like creating a project plan, it's best to use the most powerful "reasoning" model you have access to. These models are better at understanding complex requirements and breaking them down logically. You can then use a faster model for the actual step-by-step coding.
Practical Application

Let's practice and create a project plan that our AI assistant can follow. Below is a simplified example, for real projects the description of the product will be much longer and more detailed. Feel free to add more details or use your own product idea to create the plan.

Generate a project plan for a "Smart Task Manager" web application and save it in a `plan.md` file. The plan should cover: Target Audience: Who is this application for? (e.g., students, busy professionals, people learning to code). Core Problem: What simple problem does this app solve? Main Features: Add tasks via an input field. View all current tasks in a list. Mark tasks as "complete," which visually distinguishes them. Delete tasks from the list. Store tasks in the browser's local storage to persist between sessions. Tech Stack: Define this as HTML, CSS, and vanilla JavaScript.

After running this prompt, your AI will create the `plan.md` file. Open it and review the milestones. A good "Milestone 1" would be "Create the basic HTML layout with an input field, a button, and an empty list." This is a perfect, bite-sized task. You have successfully created your plan!

Outcome of This Lesson and Next Steps

You have now learned the most powerful technique for improving your AI-assisted development workflow. You've created a project plan that will serve as your guide and your AI's external memory.

This document is your defense against context drift, confusion, and wasted effort. With this plan in hand, you are ready to start building with confidence.
