In this lesson we will explore how the Agent-to-Agent (A2A) protocol and the Model Context Protocol (MCP) work together to power collaborative networks of agents. While A2A defines inter-agent communication, MCP governs the intra-agent tools and context available to each individual agent.

Why Two Protocols?

Imagine a "Research Agent" that receives a task from a "Writer Agent" via A2A. To fulfill that task, the Research Agent must access its own knowledge base and specialised tools. MCP provides the standard interface to those internal capabilities, while A2A handles the external hand-off of the task and result.

  • A2A – Inter-Agent: discovery, task negotiation, result delivery.
  • MCP – Intra-Agent: tool invocation, private data access, long-running workflows.

By cleanly separating these concerns we get a modular architecture where any agent can choose whatever internal MCP tools it likes, yet still speak the same A2A "language" to its peers.

Example Flow
  1. The Writer Agent sends a research_topic task (A2A).
  2. The Research Agent uses its MCP server to search the web, process PDFs, and summarise findings.
  3. When finished, the Research Agent packages the summary into an artifact and returns it via A2A.

This pattern can be repeated and chained, enabling sophisticated multi-step workflows across heterogeneous agents.
