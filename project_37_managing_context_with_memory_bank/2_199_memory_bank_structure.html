The Six Core Files

Memory Bank consists of six markdown files organized in a hierarchical structure:


Let's examine each file:

  1. projectbrief.md
  2. The foundation document
  3. Defines core requirements and goals
  4. Serves as the source of truth for project scope
  5. Created first and rarely changes

  6. productContext.md
  7. The "why" of your project
  8. Explains why the project exists
  9. Describes user needs and problems solved
  10. Outlines user experience goals

  11. systemPatterns.md
  12. The "how" of your project
  13. Documents system architecture
  14. Records key technical decisions
  15. Describes design patterns and component relationships

  16. techContext.md
  17. The tools and technologies
  18. Lists technologies and frameworks used
  19. Documents development setup
  20. Records technical constraints

  21. activeContext.md
  22. The current state
  23. Documents what you're working on right now
  24. Records recent changes and next steps
  25. Updated most frequently

  26. progress.md
  27. The project status
  28. Tracks completed and in-progress features
  29. Lists known issues
  30. Records the evolution of project decisions

How These Files Work Together

When you start a new session with Cursor, you'll ask it to read your Memory Bank files. Cursor will process these files in sequence, building a comprehensive understanding of your project.

Here's how the information flows:

  1. Project Brief → Establishes the foundation
  2. Product Context → Adds user-focused perspective
  3. System Patterns → Provides technical architecture
  4. Tech Context → Details specific technologies
  5. Active Context → Focuses on current work
  6. Progress → Adds historical perspective

This hierarchical approach ensures that Cursor has both the big picture and the specific details needed to assist you effectively.

Creating Your First Memory Bank

Let's walk through creating a Memory Bank for a simple project. We'll use a todo list application called "TodoMaster" as our example.

First, create a folder called "memory-bank" in your project directory. This will store all six files.

Start by creating the project brief:

# TodoMaster Project Brief ## Overview TodoMaster is a simple, intuitive task management application designed to help users organize their daily activities. ## Goals - Create a clean, intuitive interface - Make task management quick and efficient - Help users stay organized ## Scope - Web-based application - Single-user focus - Local storage for data persistence - NOT included: user accounts, sharing features ## Success Criteria - Users can manage tasks in under 3 clicks - All data persists between sessions - Interface works on both desktop and mobile

After creating the project brief, we can ask Cursor to help us create the remaining files:

I've created a projectbrief.md file for my TodoMaster application. Based on this brief, can you help me create the remaining five Memory Bank files (productContext.md, systemPatterns.md, techContext.md, activeContext.md, and progress.md)?
Key Benefits of This Structure

This hierarchical structure provides several benefits:

  1. Separation of Concerns: Each file has a clear, specific purpose
  2. Progressive Detail: Information flows from general to specific
  3. Targeted Updates: You can update specific aspects without changing everything
  4. Scalability: Works for projects of any size
