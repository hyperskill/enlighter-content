Create a file at .cursor/rules/memory-bank/systemPatterns.md with the following content: --- description: System Patterns for TodoMaster globs: *.js, *.html, *.css alwaysApply: true --- # System Patterns: TodoMaster ## Architecture Overview TodoMaster follows a simple MVC (Model-View-Controller) architecture with vanilla JavaScript. All data is stored in the browser's localStorage. ## Component Structure - **Models**: Handle data management and business logic - TaskModel: Manages task data (CRUD operations) - CategoryModel: Manages category data - StorageModel: Handles localStorage interactions - **Views**: Handle UI rendering - TaskListView: Renders the list of tasks - TaskFormView: Renders task creation/editing form - FilterView: Renders filtering controls - **Controllers**: Connect models and views - AppController: Main application controller - TaskController: Handles task-related operations - FilterController: Manages filtering logic ## Design Patterns - **Repository Pattern**: For data access abstraction - **Observer Pattern**: For UI updates when data changes - **Module Pattern**: For code organization - **Factory Pattern**: For creating task and category objects ## Data Flow 1. User interacts with a View 2. View notifies Controller of the action 3. Controller updates Model as needed 4. Model updates data in localStorage 5. Model notifies Controller of changes 6. Controller updates View to reflect changes

This file provides a technical blueprint for our app, including:

  • The architecture (MVC with localStorage)

  • The design patterns we'll use
  • How data flows through the system
